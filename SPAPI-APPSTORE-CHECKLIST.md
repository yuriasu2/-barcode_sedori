# SP-API 公開アプリ化 チェックリスト（Appstore掲載・ウェブサイトガイドライン）

作成: 2026-07-17 / 出典: Amazon公式ドキュメント（website-guidelines / list-your-app-on-the-selling-partner-appstore、2026-07時点の全文を確認）

## 0. 重要な前提知識

- **掲載は「実は」必須ではないが、未掲載アプリには認可数の上限（Authorization Limits）がかかる**。
  公式: "Although you're not required to list your app on the Selling Partner Appstore, authorization limits apply to unlisted apps that serve sellers."
  → 少人数のβ運用は掲載前でも可能。ユーザー数を伸ばすなら掲載が実質必須。
- 掲載審査の所要: **承認後、公開まで3〜4週間**。不承認時は違反点が通知され再申請可。
- 現在のOAuth実装は `version=beta`（ドラフト用）。掲載承認後は beta 無しの本番URLに切替える。

## 1. sellira.jp ウェブサイトガイドライン チェックリスト

### アクセシビリティ・機能
- [ ] 常時アクセス可能（「準備中」表示・プレースホルダ文言・セキュリティ警告がない）
- [ ] ログイン不要でアプリ情報を閲覧できる
- [ ] 自前サイトである（他のアプリストアの一部でない）✔
- [ ] 十分なテキスト量がある（画像だけのページにしない）
- [ ] リンク切れ・低解像度画像がない

### 事業者情報・連絡先
- [ ] **会社名/開発者名・アプリ名・ロゴを明示**（開発者プロファイルの名義と一致させる）
- [ ] **直接の連絡手段を最低1つ**（メールアドレス / 電話 / 問い合わせフォーム。SNSリンクだけでは不可）

### セキュリティ
- [x] HTTPS + 有効な証明書 ✔（Cloudflare）
- [ ] 過剰なポップアップ・広告・他サイトへのリダイレクトがない
- [ ] **包括的なプライバシーポリシー**（データの収集・利用・保存・保護・共有・削除を明記。AmazonのDPPおよび日本の法令に準拠）← **最重要・要新規作成の可能性大**

### 製品情報
- [ ] **機能の包括的な説明**（申請したロール=Catalog/Pricing/Fees の利用目的と整合させる）
- [ ] **料金体系の明示**（無料プランの内容と Pro ¥1,980/月・3日間無料トライアルを明記）

### ブランド・マーケティング表現
- [ ] Amazonの商標ガイドライン順守（「AMZ」含むAmazon商標をアプリ名/URL/ロゴ等に使わない）
- [ ] アプリ名が一般名詞のみでない（「Sellira」ブランド名付きなので ✔）
- [ ] **「No.1」「最強」「唯一」等の未検証の優位性主張をしない**（LPの文言を要総点検）
- [ ] Amazonとの提携を示唆する表現をしない

### ポリシー
- [ ] AUP / DPP / Solution Provider Agreement に準拠（DPP対応の残件: refresh tokenのKeychain移行 等）

## 2. Appstore掲載フォーム 準備物リスト

申請場所: Solution Provider Portal（またはSeller Central > アプリと サービス > アプリ開発）→ 対象アプリの「Create listing」

### テキスト（禁則: `"` `'` `&` `<` `>` 使用不可。`$`は`USD`と書く。「最高/No.1/唯一」等の表現不可）
- [ ] 言語: 日本語
- [ ] リスティング名（アプリ名）
- [ ] カテゴリ: **Product Research and Scouting**（商品リサーチ・仕入れ判断）が最適
- [ ] 短い説明（**200字以内**）
- [ ] 長い説明（**4000字以内**。事実ベース・カテゴリと整合）
- [ ] アプリ機能の箇条書き（**1000字以内**）
- [ ] 対象ユーザー説明（**1000字以内**。日本のAmazonストア限定・iOS必須などの要件を明記）
- [ ] Product URL: `https://sellira.jp`（サインインページ不可・https必須）
- [ ] Support URL（ヘルプ/サポートページ。sellira.jp内に用意）
- [ ] サポートメールアドレス
- [ ] サポート電話番号
- [ ] サポート対応言語: 日本語
- [ ] Functionality / Programs supported / **Amazon stores supported: Japan**

### 料金ページ
- [ ] Pricing model: Starting at ¥→**USD換算注意**（$表記不可・USDと記載）または Free+説明
- [ ] Free trial: 3日 を Duration に記載
- [ ] Pricing description に無料プラン+Pro月額の構成を説明

### 画像（共通: ほぼ正方形・2MB以下・Amazonロゴ/商標不可）
- [ ] 詳細ページ用ロゴ: **300×300px以上**
- [ ] カテゴリページ用ロゴ: **220×220px以上**（詳細ページと同一推奨）
- [ ] 追加画像（任意・最大6枚）: アプリのスクリーンショット等、300×300以上の正方形
- [ ] バナー（任意・最大2枚）: 970×300px以上

### 提出後
- [ ] ケースログで進捗確認（承認→3〜4週間で公開 / 不承認→違反点の通知→修正して再申請）

## 3. 掲載承認後にやること（コード側）

- [ ] `server/src/oauth.js` の認可URLから `version=beta` を外す（環境変数化して切替可能にしておくと再デプロイ不要）
- [ ] 本番OAuthの通し確認（第三者セラーの連携）

## 4. 推奨順序

1. sellira.jp のガイドライン適合（プライバシーポリシー新設・料金/連絡先/機能説明の整備・文言点検）
2. （並行）DPP残件: refresh token の Keychain 移行
3. 掲載フォームの素材準備（ロゴ画像・説明文）→ 申請
4. 審査中: `version=beta` の環境変数化・実機のβテスト（未掲載でも認可上限内で可能）
5. 承認後: beta解除 → 一般公開
