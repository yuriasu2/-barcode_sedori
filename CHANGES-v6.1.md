# 改修仕様 v6.1 — グラフ期間切替 / カード余白調整 / 商品タブのキャッシュ表示 (2026-07-05)

## 1. Keepaグラフの期間切替 (90日/1年/3年)

### サーバー (routes.jsのgraphハンドラ)
- `GET /api/graph?asin=&range=` の `range` クエリを受け付ける。許可値は **90 / 365 / 1095** のみ(それ以外・未指定は90)
- Keepaへのgraphimageリクエストの `range` に反映
- グラフキャッシュのキーに range を含める(期間別にキャッシュ)
- テスト: range検証(不正値→90扱い)とキャッシュキー分離を test に追加

### iOS
- グラフ画像の直下にセグメント切替「90日 / 1年 / 3年」(初期値90日)を配置
- 切替で `/api/graph?asin=&range=` のURLを変えてAsyncImageを再ロード
- APIClient.graphURL(asin:range:) に拡張

## 2. 検索画面の結果カードの余白・枠削除 (iOS)

対象: 商品画像・「本」バッジ・タイトル・コード・ランキングを表示しているカード部分
- カードの**上下の余白(padding/spacing)を0**にする(左右は現状維持でよい)
- カードの**薄い灰色の囲み枠(background/border/角丸ボックス)を削除**し、その枠に伴う上下マージンも0にする
- 目的: カメラ〜オファーパネル〜グラフを1画面に収める縦方向の圧縮

## 3. 商品タブ(履歴)のキャッシュ表示 (iOS)

現状: 履歴の商品をタップすると ProductDetailView が /api/offers を再度呼んでいる。
変更:
- **スキャン時に取得済みのデータをそのまま表示し、APIを再度呼ばない**
- ScanHistoryItem に、検索結果(SearchResult)に加えて**オファー一覧(OffersResult)も保存**する。
  検索タブで第2段階(offers)の取得が完了した時点で、該当履歴エントリを更新して保存する
  (offersが未取得のまま履歴入りした場合は「価格一覧は未取得です」表示。API呼び出しはしない)
- 履歴からの詳細画面は保存済みデータのみで描画する専用表示(既存ProductDetailViewを
  「渡されたデータで描画するモード」に改修するか、履歴用ビューを作るかは実装判断。API呼び出しコードパスに入らないこと)
- **商品情報セクションにJANコード(またはISBN-13)の行を追加**(ラベル「JANコード」、値は isbn13 ?? スキャンコード)
- 既存の保存データ(旧形式のScanHistoryItem)がデコードできず履歴が消えないよう、新フィールドはOptionalにして後方互換を保つ
